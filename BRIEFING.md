## 1. 한 줄 요약

사우디아라비아 6개 도시 **랜드마크 40곳**을 지도앱용 **3D 리소스로 제작**하기 위한
**모델링 레퍼런스 + 작업물 공유 사이트**. 정적 웹사이트(빌드 후 GitHub Pages 호스팅).

- 대상: Riyadh, Jeddah, Al Khobar, Dammam, Makkah, AlUla
- Tier 분류: **T1(핵심) 10 / T2(주요) 25 / T3(추가) 5**

## 2. 페이지 구성 (2개)

| 페이지 | URL | 역할 | 사용자 |
|---|---|---|---|
| **Reference** | `/` | 랜드마크별 모델링 참고 자료 (스펙시트) | 모델링 작업자 |
| **Files** | `/progress/` | 랜드마크별 3D 모델·이미지 다운로드 | 클라이언트·검수자 |

상단 탭으로 전환. 다크모드 토글 있음(상태 기억).

### Reference 페이지 — 무엇이 들어있나
- 상단: **납품 사양 카드**(접기/펼치기) + **Glossary**(건축 용어 42개, 본문 호버 툴팁)
- 사이드바: Tier/City/Type/**3D 모델 보유** 필터 + 검색(이름·구조·핵심요소까지) + Index
- 카드별: 이미지 슬라이더 + 필수요소 / Key Points / Structure / Part Heights
- 밀도 토글: **Detailed / Compact**
- 모델 있는 카드엔 `3D Viewer` 버튼 → 공유 3D 모달

### Files 페이지 — 무엇이 들어있나
- 랜드마크별 카드 그리드(번호순). 현재 **8개 업로드됨**(01·02·03·04·06·07·08·13)
- 카드: 커버 이미지 + `3D Viewer` 버튼 + .glb 다운로드 + 이미지 썸네일
- **로그·날짜 없음** — 항상 최신 파일만. 받는 사람은 즉시 다운로드만

### 공유 3D 뷰어 (양쪽 페이지 공통 모달)
- 회전·확대·이동 (model-viewer 기반)
- **삼각형(tris) 수** 표시 (glb에서 빌드 시 정확 계산)
- **와이어프레임 토글** (메시 토폴로지 검수)
- 모델 다운로드 / 여러 버전 시 탭 전환

## 3. 구축 방식 (기술)

**정적 사이트 — 빌드 후 결과물만 호스팅. 백엔드/DB 없음.**

```
data/*.json + templates/*.tpl + images/ + progress/
            │
   python3 scripts/build.py   ← 단일 빌드 스크립트
            │
          docs/  (index.html + progress/ + 이미지/모델 동기화)
            │
   git push → GitHub Pages 자동 배포 (1~2분)
```

- **순수 HTML/CSS/JS** (프레임워크 없음). 단일 self-contained HTML로 빌드
- **3D 뷰어**: Google `<model-viewer>` (CDN, v3.5.0 고정)
- **폰트**: Inter + Noto Sans KR + JetBrains Mono(기술 토큰용)
- **디자인 톤**: iOS 도큐먼트 스타일 — 한색(쿨 그레이) 팔레트, 헤어라인 리스트

### 폴더 구조
| 폴더 | 내용 |
|---|---|
| `data/` | landmarks.json(40개 데이터), glossary.json(용어) |
| `templates/` | index.html.tpl(Reference), progress.html.tpl(Files) |
| `images/` | 레퍼런스 카드 이미지 (`{번호}_{슬러그}.jpg`) |
| `progress/` | 업로드한 3D 모델·작업 이미지 (`KSA-NN.glb`, `NN-1.png`) |
| `guides/` | 카드별 모델링 가이드 문서 (현재 02·06·13) |
| `docs/` | **빌드 결과물** (GitHub Pages가 서빙) — 직접 편집 X |
| `scripts/` | build.py(빌드), fetch_images.py(이미지 수집) |

## 4. 핵심 기준·규칙 (클라이언트 관련)

### A. 납품 사양 (사이트 상단 "납품 사양" 카드에 명시)
- **단위**: 미터(m) — 불일치 시 납품 불합격
- **좌표계**: EPSG:4326 (WGS84) 경위도, 소수점 8자리 이상
- **포맷**: `.glb` 바이너리
- **텍스처**: 기본 미포함
- **메시**: 가시 면만 · 삼각형 면 · 비가시/하단/접합 면 제외 · Non-manifold 미허용
- **Footprint**: 위탁자 제공 EPSG:3857 폴리곤 기준 위치 정합
- **폴리곤 한도**: 소형 3k / 중형 10k / 대형 20k / 초대형 50k
  - ⚠ 현재 *취소선 처리* — 실제 한도가 유동적이라 참고용으로만 표시

### B. 랜드마크별 메타 (카드 Structure에 노출)
- **폴리곤 예산**(규모 분류 기준), **WGS84 좌표**(xlsx 핀에서 추출), **시공 상태**, **작업 구분**

### C. 데이터 신뢰도 — ⚠ 미팅에서 중요
각 카드를 **3단계**로 분류 (자료 신뢰도):
- 🟢 **verified (8)**: 설계자·연도·치수를 권위 자료로 검증
- 🟡 **partial (9)**: 건물 식별 OK, 구조 치수는 추정
- 🔴 **low (23)**: 공식 자료 매우 부족 → `DATA LOW` 뱃지 표시

> **핵심 메시지**: 절반 이상(23개)이 공개 자료 부족. 정확한 footprint·치수는
> **위탁자(클라이언트) 제공 자료** 또는 위성/실측으로 채워야 함. 사이트는 이를
> 숨기지 않고 명시(DATA LOW)해서 리스크를 투명하게 드러냄.

### D. 좌표 정밀도
- 모델 자체엔 절대좌표를 안 박음 (float32 정밀도 한계)
- **로컬 원점 + 메타데이터(.json)** 분리: 원점 WGS84 + 방위각 + 오프셋 + 스케일
- 변환 툴은 **위탁자 제공 예정** (아직 미수령)

## 5. 업데이트 워크플로우 (팀원이 알아야 할 것)

### 새 3D 모델·이미지 올리기
1. `progress/` 폴더에 **번호 규칙**으로 파일 넣기
   - 모델: `KSA-01.glb` (또는 `01.glb`)
   - 이미지: `01-1.png`, `01-2.png` ...
   - 번호만 맞으면 자동으로 해당 랜드마크에 매칭 (JSON 편집 불필요)
2. `python3 scripts/build.py`
3. `git add -A && git commit -m "..." && git push`
4. 1~2분 후 라이브 반영

→ **덮어쓰면 최신본**으로 교체. Files 페이지가 폴더를 자동 스캔.

### 레퍼런스 데이터 수정
- `data/landmarks.json` 편집 → 빌드 → 푸시
- 이미지는 `images/` 폴더 (`{번호}_{슬러그}.jpg`)

## 6. 미팅 전 알아둘 포인트 (요약)

1. **사이트는 2가지 역할** — 작업 참고(Reference) + 결과물 공유(Files)
2. **현재 8개 모델 업로드됨** (40개 중). T1 위주로 진행 중
3. **데이터 신뢰도 투명 공개** — 23개는 DATA LOW, 클라이언트 자료가 정확도의 핵심
4. **footprint·변환 툴은 클라이언트 제공 대기 중** — 받으면 위치·스케일 정밀 보정
5. **3D 뷰어로 즉석 검수 가능** — 회전·tris 수·와이어프레임을 미팅 중 직접 시연
6. **정적 사이트라 안정적·무료 호스팅** — 빌드+푸시만으로 누구나 최신 확인
7. **납품 사양은 사이트에 박제** — 단위·좌표계·포맷·메시 규칙을 모두가 같은 기준으로 참조

### 시연 추천 동선 (미팅 중)
Reference에서 카드 훑기 → 납품 사양/Glossary 펼쳐 보이기 → Files 탭 →
`3D Viewer` 열어 회전·와이어프레임·tris 보여주기 → 다운로드 시연

## 7. 알려진 제약·주의

- **DATA LOW 23개**: footprint 수령 전엔 추정 기반
- **폴리곤 한도 취소선**: 현재 모델들이 한도 초과 상태 많음(텍스처 없는 형태 모델이라 tris 큼) — 최적화 단계 별도 필요
- **model-viewer 와이어프레임/tris**: 내부 API 활용 → 버전 올리면 점검 필요 (현재 v3.5.0 고정)
- **건설 중 항목**(Aramco Stadium, Six Flags Qiddiya): 위성과 최종 도면 다를 수 있음
