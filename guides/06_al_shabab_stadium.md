## 4. 블록아웃 (~30분)

### 추정 치수 (검증 전 출발점)

14k 소형 스타디움 기준 — 04 KSU(25k, ~230×200×22m)의 약 **70-80%**:

| 부위 | 사이즈 | 비율 |
|---|---|---:|
| 전체 외곽 | ~180 × 150 m | — |
| 총 높이 | ~18 m | 100% |
| 기둥/필로티 | ~4 m | 22% |
| 콘코스 외장 | ~5 m | 28% |
| 객석 매스 | ~7 m | 39% |
| 지붕 (있다면) | ~2 m | 11% |

### Phase A — 매스 블록아웃 (~150 tris)

**파라메트릭 유지 (절대 `C` 누르지 말 것)**

```
KSA_06_Al_Shabab (Null)
├── pilotis    [Cube]  180 × 4 × 150m   @ P.Y=2     ← 지상 기둥층
├── concourse  [Cube]  180 × 5 × 150m   @ P.Y=6.5   ← 콘코스 외장
├── bowl       [Tube]  외경 180, 내경 110, 높이 7  @ P.Y=12.5  ← 객석 외형
│              (디스크 세그먼트로 오벌 모양 잡거나 X·Z 비율로 압축)
└── roof       [Cube 또는 생략]  180 × 2 × 30m  @ P.Y=17  
               ← 메인 그랜드스탠드(서측) 부분 지붕만, 없으면 생략
```

**중요 결정 — 지붕 유무**:
- 위성뷰 탑뷰에서 그라운드(필드)가 그늘 없이 다 보이면 → 지붕 없음, `roof` 매스 생략
- 한쪽만 그늘이 있으면 → 그쪽 메인 스탠드만 `roof` 추가
- 둘레 다 그늘이면 → 풀 캐노피 (전체 둘레 부분 지붕)

### Phase B — 외장 패널 (~3,000 tris, 선택)

알샤밥 클럽 컬러 식별 단서가 외장에 있을 수 있음 (백/검정 띠).
- Cloner로 수직 패널 반복 배치 (필요 시)
- 자료 부족 — 위성·사진 보고 패턴 결정
- **블록아웃 단계엔 생략 가능**, 디테일 단계로 미루기

### Phase C — 검토 · 클린업 (~5분)

- [ ] Object Manager: 모든 객체 파라메트릭 (Cube/Tube/Cloner 아이콘)
- [ ] Editable(▼) 아이콘 없음
- [ ] Null로 그룹화
- [ ] 명명 일관 (`pilotis`, `concourse`, `bowl`, `roof`)
- [ ] 비가시 면 — 보울 내부 바닥, 인접 매스 접합면
